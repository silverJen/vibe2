## 1. 물 섭취 기록 기능

### US-001: 간단한 물 섭취 기록
**As a** 바쁜 직장인  
**I want to** ml 단위 입력 없이 간단하게 물 섭취를 기록하고 싶다  
**So that** 복잡한 입력 과정 없이 빠르게 기록할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 사용자는 `마셨음`, `조금 마셨음`, `거의 안 마셨음` 중 하나를 선택할 수 있다
- [ ] 정확한 ml 단위 입력은 요구하지 않는다
- [ ] 버튼 한 번의 클릭으로 기록이 완료된다
- [ ] 기록 완료 시 간단한 피드백(애니메이션 또는 메시지)을 제공한다

**우선순위:** P0 (MVP 필수)

---

### US-002: 기록 누락 허용
**As a** 수험생  
**I want to** 기록을 하지 않은 날에 대해 죄책감을 느끼지 않고 싶다  
**So that** 스트레스 없이 앱을 지속적으로 사용할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 하루에 0회 기록도 허용된다
- [ ] 연속 기록(streak) 개념이 없다
- [ ] 기록하지 않은 날은 "데이터 없음"으로 처리된다
- [ ] 기록 누락에 대한 알림이나 경고를 표시하지 않는다
- [ ] 기록 누락 후에도 앱 재방문 시 긍정적인 메시지를 표시한다

**우선순위:** P0 (MVP 필수)

---

### US-003: 다중 기록 지원
**As a** 사용자  
**I want to** 하루에 여러 번 물 섭취를 기록하고 싶다  
**So that** 실제 물 섭취 패턴을 정확하게 반영할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 하루에 n회 기록이 가능하다
- [ ] 각 기록은 시간과 함께 저장된다
- [ ] 같은 날 여러 번 기록해도 중복 경고가 없다
- [ ] 기록 목록에서 각 기록의 시간을 확인할 수 있다

**우선순위:** P0 (MVP 필수)

---

### US-004: 기록 수정 및 삭제
**As a** 사용자  
**I want to** 잘못 입력한 기록을 수정하거나 삭제하고 싶다  
**So that** 정확한 데이터를 유지할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 기록한 내용을 수정할 수 있다
- [ ] 기록을 삭제할 수 있다
- [ ] 수정/삭제 시 확인 메시지를 표시한다
- [ ] 수정/삭제 후 즉시 화면에 반영된다

**우선순위:** P1 (경험 강화)

## 2. 물 섭취 조회 기능

### US-005: 일별 기록 조회
**As a** 사용자  
**I want to** 오늘 내가 마신 물의 기록을 한눈에 보고 싶다  
**So that** 오늘 얼마나 물을 마셨는지 확인할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 홈 화면에서 오늘의 물 섭취 기록을 확인할 수 있다
- [ ] 기록된 시간과 선택한 항목(`마셨음` 등)을 표시한다
- [ ] 기록이 없는 경우 "아직 기록이 없어요"와 같은 친절한 메시지를 표시한다
- [ ] 간단한 시각적 표현(아이콘, 색상 등)을 사용한다

**우선순위:** P0 (MVP 필수)

---

### US-006: 주간 기록 조회
**As a** 사용자  
**I want to** 지난 일주일 동안의 물 섭취 패턴을 보고 싶다  
**So that** 내 습관 변화를 파악할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 최근 7일간의 물 섭취 기록을 시각화하여 표시한다
- [ ] 요일별로 구분하여 확인할 수 있다
- [ ] 기록이 없는 날은 명확하게 표시한다
- [ ] 간단한 패턴(요일별 경향성 등)을 시각적으로 확인할 수 있다

**우선순위:** P0 (MVP 필수)

---

### US-007: 월간 기록 조회
**As a** 사용자  
**I want to** 지난 한 달간의 물 섭취 기록을 보고 싶다  
**So that** 장기적인 변화 추이를 확인할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 최근 30일간의 기록을 캘린더 형태로 확인할 수 있다
- [ ] 각 날짜별로 물 섭취 정도를 색상이나 아이콘으로 구분한다
- [ ] 특정 날짜를 클릭하면 상세 기록을 볼 수 있다
- [ ] 월별 전환이 가능하다

**우선순위:** P1 (경험 강화)

---

### US-008: 시간대별 패턴 조회
**As a** 사용자  
**I want to** 내가 주로 어느 시간대에 물을 마시는지 알고 싶다  
**So that** 물 섭취 습관에 대한 인사이트를 얻을 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 시간대별(아침, 점심, 저녁, 밤 등) 물 섭취 빈도를 표시한다
- [ ] 최소 1주일 이상의 데이터를 기반으로 분석한다
- [ ] 간단한 차트나 그래프로 시각화한다
- [ ] "주로 오후에 물을 많이 마셔요"와 같은 설명을 제공한다

**우선순위:** P1 (경험 강화)

## 3. AI 기반 리포트 기능

### US-009: 주간 AI 해석 리포트 생성
**As a** 사용자  
**I want to** AI가 내 물 섭취 패턴을 분석하여 의미 있는 해석을 제공받고 싶다  
**So that** 단순한 숫자가 아닌 나의 변화를 이해할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 최소 3일 이상의 데이터가 있을 때 리포트가 생성된다
- [ ] 주 1회 자동으로 리포트가 생성된다
- [ ] Gemini API를 사용하여 자연어 리포트를 생성한다
- [ ] 리포트는 "평가·훈계"를 하지 않고 "관찰 → 해석 → 가벼운 제안" 순서로 구성된다
- [ ] 예시: "완벽하진 않았지만, 지난주보다 오후 물 섭취 빈도가 늘었어요."

**우선순위:** P0 (MVP 필수)

---

### US-010: 주문형 리포트 생성
**As a** 사용자  
**I want to** 필요할 때 즉시 AI 리포트를 요청하고 싶다  
**So that** 현재 상태를 바로 확인할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] "리포트 생성" 버튼을 통해 즉시 리포트를 요청할 수 있다
- [ ] 최소 3일 이상의 데이터가 있어야 생성 가능하다
- [ ] 데이터가 부족한 경우 친절한 안내 메시지를 표시한다
- [ ] 생성 중 로딩 상태를 표시한다
- [ ] 생성 완료 후 즉시 조회할 수 있다

**우선순위:** P0 (MVP 필수)

---

### US-011: 리포트에서 컨디션 연관성 제공
**As a** 미용·컨디션 개선에 관심 있는 사용자  
**I want to** 물 섭취와 내 컨디션(피로, 붓기 등)의 연관성을 알고 싶다  
**So that** 물 섭취가 내 몸에 어떤 영향을 주는지 이해할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 물 섭취 데이터와 컨디션 메모를 함께 분석한다
- [ ] 패턴을 발견한 경우 자연어로 설명한다
- [ ] 예시: "물이 적었던 날에 피로 메모가 자주 등장했어요."
- [ ] 단정적 표현 대신 가능성과 패턴 언어를 사용한다
- [ ] 컨디션 데이터가 없어도 물 섭취 패턴만으로 리포트를 생성한다

**우선순위:** P0 (MVP 필수)

---

### US-012: 리포트 히스토리 조회
**As a** 사용자  
**I want to** 과거에 생성된 AI 리포트를 다시 보고 싶다  
**So that** 시간에 따른 변화를 비교하고 확인할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 생성된 모든 리포트를 목록으로 확인할 수 있다
- [ ] 리포트는 생성일 기준으로 정렬된다
- [ ] 각 리포트를 클릭하여 전체 내용을 볼 수 있다
- [ ] 리포트는 Supabase Storage에 저장된다

**우선순위:** P2 (MVP 이후 고려)

---

### US-013: 리포트 톤 설정
**As a** 사용자  
**I want to** AI 리포트의 톤(공감적, 격려형, 분석형 등)을 선택하고 싶다  
**So that** 내가 선호하는 스타일로 리포트를 받을 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 리포트 톤을 선택할 수 있는 설정 메뉴가 있다
- [ ] 최소 3-5가지 톤 옵션을 제공한다
- [ ] 선택한 톤이 다음 리포트 생성에 반영된다
- [ ] 기본값은 "공감적, 설명 중심"으로 설정된다

**우선순위:** P2 (MVP 이후 고려)

---

### US-014: 데이터 부족 시 리포트 대응
**As a** 사용자  
**I want to** 데이터가 부족해도 AI가 이를 긍정적으로 설명해주길 바란다  
**So that** 부족함 자체로 실패감을 느끼지 않을 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 데이터가 부족한 경우에도 리포트를 생성한다
- [ ] "기록이 적었지만, 괜찮아요"와 같은 긍정적 메시지를 포함한다
- [ ] 부족함을 실패가 아닌 상황으로 설명한다
- [ ] 간단한 제안(부담 없는 수준)을 제공한다

**우선순위:** P0 (MVP 필수)

## 4. 컨디션 기록 기능

### US-015: 일일 컨디션 메모
**As a** 사용자  
**I want to** 오늘의 컨디션(피로, 붓기 등)을 간단하게 기록하고 싶다  
**So that** AI 리포트에서 물 섭취와 컨디션의 연관성을 확인할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 하루 최대 1회, 간단한 질문으로 컨디션을 물어본다
- [ ] 예시: "오늘 컨디션은 어땠어?", "붓기나 피로를 느꼈어?"
- [ ] 선택형 버튼으로 답변할 수 있다
- [ ] 기록을 강요하지 않으며, 건너뛰기가 가능하다
- [ ] 기록한 컨디션은 물 섭취 기록과 연결되어 저장된다

**우선순위:** P1 (경험 강화)

---

### US-016: 컨디션 조회
**As a** 사용자  
**I want to** 과거에 기록한 컨디션 메모를 확인하고 싶다  
**So that** 내 컨디션 변화를 추적할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 물 섭취 기록 화면에서 컨디션 메모를 함께 확인할 수 있다
- [ ] 달력 또는 리스트 형태로 조회 가능하다
- [ ] 특정 날짜의 컨디션 상세 내용을 볼 수 있다

**우선순위:** P1 (경험 강화)

## 5. 알림 기능

### US-017: 맥락 기반 알림 설정
**As a** 사용자  
**I want to** 맥락을 고려한 알림을 원하는 시간대에만 받고 싶다  
**So that** 집중을 방해받지 않으면서도 물 섭취를 상기할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 알림은 기본적으로 꺼져 있다 (Opt-in)
- [ ] 사용자가 원하는 시간대를 지정할 수 있다 (예: 오후 3-5시)
- [ ] 알림 톤은 권유형/허용형이다 (명령형 X)
- [ ] 예시: "지금 한 모금 정도면 충분해요"
- [ ] 알림 빈도를 설정할 수 있다

**우선순위:** P1 (경험 강화)

---

### US-018: 리포트 생성 알림
**As a** 사용자  
**I want to** 새로운 AI 리포트가 생성되면 알림을 받고 싶다  
**So that** 리포트를 놓치지 않고 확인할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 주간 리포트가 생성되면 푸시 알림을 전송한다
- [ ] 알림을 누르면 리포트 화면으로 이동한다
- [ ] 알림 설정에서 이 알림을 끄고 켤 수 있다
- [ ] 알림 문구는 긍정적이고 호기심을 유발한다

**우선순위:** P1 (경험 강화)

## 6. 기타 기능

### US-019: 온보딩 프로세스
**As a** 신규 사용자  
**I want to** 앱의 핵심 가치를 간단하게 이해하고 시작하고 싶다  
**So that** 앱을 어떻게 사용해야 할지 명확히 알 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 최소한의 설정만 요구한다
- [ ] 앱의 차별점("의미 해석")을 명확히 전달한다
- [ ] "실패 없음" 개념을 강조한다
- [ ] 2-3개 화면 정도의 간단한 튜토리얼을 제공한다
- [ ] 건너뛰기 옵션이 있다

**우선순위:** P0 (MVP 필수)

---

### US-020: 다크 모드 지원
**As a** 사용자  
**I want to** 다크 모드를 사용하고 싶다  
**So that** 눈의 피로를 줄이고 밤에도 편하게 사용할 수 있다

**인수 기준 (Acceptance Criteria)**
- [ ] 라이트/다크 모드를 전환할 수 있다
- [ ] 시스템 설정을 따를 수 있다
- [ ] 모든 화면이 다크 모드를 지원한다
- [ ] 색상 대비가 적절하여 가독성이 유지된다

**우선순위:** P2 (MVP 이후 고려)
