# 📄 PRD – 물 섭취를 ‘의미 있는 변화’로 해석하는 습관형 앱

---

## 1. 제품 개요 (Product Overview)

### 1.1 Value Positioning (한 문장)
> **우리 제품은 바쁜 일상 속에서 물을 마셔야 한다는 건 알지만 관리할 여유는 없는 사람들이, 숫자 입력·잦은 알림 중심의 기존 물 섭취 앱보다, 아무 노력 없이도 물 섭취 기록을 생활 변화와 컨디션으로 해석해줘서 ‘그래서 뭐가 달라졌는지’를 이해하게 돕는다.**

### 1.2 문제 정의 (Problem Statement)
- 사용자는 이미 “물을 마셔야 한다”는 사실을 알고 있음
- 기존 물 섭취 앱의 한계:
  - ml 입력, 앱 실행 자체가 피로
  - 상황을 고려하지 않는 알림
  - 기록 실패 시 죄책감 유발
  - 기록 결과가 몸·생활 변화로 연결되지 않음
- 결과적으로 **2~4주 내 자연 이탈**이 반복됨

### 1.3 타겟 사용자 (Primary Target)
- 20–39세 바쁜 직장인
- 집중을 방해받기 싫은 수험/공부 사용자
- 미용·컨디션 개선에 관심 있는 사용자
- 공통 특징:
  - 기록 의지가 낮음
  - 실패 경험에 민감
  - “해석”과 “이유”를 원함

---

## 2. 목표와 비목표 (Goals & Non-Goals)

### 2.1 제품 목표 (Goals)
1. 기록에 대한 **인지적 부담 제거**
2. **실패하지 않는 경험** 제공
3. 물 섭취 데이터를 **의미 있는 변화로 해석**
4. 앱을 자주 열지 않아도 가치를 느끼게 함

### 2.2 비목표 (Non-Goals)
- 정확한 ml 단위 수분 섭취 측정
- 게임/캐릭터/경쟁 중심 동기부여
- 의료·치료 목적의 수분 관리

---

## 3. 사용자 인사이트 요약 (Interview Insights)

| 인사이트 | 근거 페르소나 |
|--------|--------------|
| 기록은 습관이 아니라 부담으로 인식 | 김지혜 |
| 알림은 맥락 없으면 방해가 됨 | 박성훈 |
| 숫자는 변화로 연결되지 않음 | 정하린 |
| 불투명한 유료 전환은 즉시 이탈 | 최민준 |

**핵심 인사이트**
> 사용자는 “더 열심히 기록”이 아니라  
> **“신경 쓰지 않아도 괜찮은 상태”**를 원함

---

## 4. Core Gap 정의

> ❗ **기록을 잘하게 만드는 문제는 이미 해결되었다.  
하지만 기록을 ‘의미 있는 변화’로 설명해주는 문제는 거의 해결되지 않았다.**

---

## 5. 핵심 사용자 시나리오 (Core User Journey)

1. 앱 설치 → 최소 설정만 진행
2. 하루 중 0~1회, 매우 간단한 선택형 입력
3. 기록 누락일 발생 → 실패 처리 없음
4. 일정 데이터 누적 후:
   - AI가 패턴과 변화를 **언어로 해석**
5. 사용자는:
   - “나는 실패하지 않았다”고 느끼고
   - 다시 앱을 켤 이유를 가짐

## 6. 기능 우선순위 (Feature Prioritization)

### 6.1 P0 – MVP 필수 기능

#### 1) 무부담 물 섭취 기록
- ml 숫자 입력 없음
- 버튼 선택형 기록:
  - `마셨음 / 조금 마셨음 / 거의 안 마셨음`
- 하루 기록 0회도 허용

#### 2) 실패 개념 제거
- 연속 기록(streak) 없음
- 누락일은 “데이터 없음”으로 처리
- 기록 안 한 날에 대한 알림/경고 없음

#### 3) AI 기반 ‘의미 해석 리포트’ (핵심 차별점)
- Gemini API 연동
- 입력 데이터:
  - 물 섭취 패턴
  - 요일/시간대
  - (선택) 컨디션/피로/붓기 메모
- 출력 원칙:
  - 평가·훈계 금지
  - 실패 전제 금지
  - 관찰 → 해석 → 가벼운 제안 순서

**출력 예**
- “완벽하진 않았지만, 지난주보다 오후 물 섭취 빈도가 늘었어요.”
- “물이 적었던 날에 피로 메모가 자주 등장했어요.”

---

### 6.2 P1 – 경험 강화 기능

#### 4) 맥락 기반 알림 (Opt-in)
- 기본값: 알림 OFF
- 설정 시:
  - 시간대만 지정 (예: 오후 3–5시)
  - 톤: 권유형 / 허용형
  - 예: “지금 한 모금 정도면 충분해요”

#### 5) 해석 보조 질문
- 하루 최대 1회
- 기록 강요 아님
- 예:
  - “오늘 컨디션은 어땠어?”
  - “붓기나 피로를 느꼈어?”

---

### 6.3 P2 – MVP 이후 고려
- 주간 요약 공유
- 홈 위젯
- 해석 리포트 보관함

## 8. 기술 스택 (요청 범위 내)

### 8.1 프론트엔드 / 백엔드
- **Next.js**
  - App Router 기반
  - 서버/클라이언트 통합 구조

### 8.2 데이터 / 인증
- **Supabase**
  - Auth: 이메일 기반 인증
  - DB: 사용자, 물 섭취 기록, 컨디션 메모, AI 리포트
  - Storage: 생성된 리포트 텍스트 저장

### 8.3 AI
- **Gemini API**
  - 물 섭취 + 컨디션 데이터를 자연어로 해석
  - 주간/비정기 리포트 생성
